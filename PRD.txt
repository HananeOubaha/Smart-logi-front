Contexte du projet
L’application SmartLogi Delivery Management System (SDMS) dispose désormais d’un backend robuste, sécurisé et industrialisé, comprenant :

API REST Spring Boot
Sécurité stateless JWT
Authentification OAuth2 (Social Login)
Déploiement Docker
Pipeline CI/CD avec Jenkins
Cependant, l’application ne dispose pas encore d’une interface utilisateur moderne permettant aux différents acteurs métiers d’interagir efficacement avec le système.

SmartLogi souhaite donc développer une application frontend web afin de :

offrir une interface claire et intuitive aux utilisateurs,
consommer les APIs sécurisées existantes,
respecter les rôles et permissions définis côté backend,
préparer l’intégration future avec des clients web et mobiles.
L’application frontend sera développée avec Angular, en suivant les bonnes pratiques de structuration, de sécurité et de maintenabilité.

Objectifs métier
L’interface Angular devra permettre :

Une authentification sécurisée des utilisateurs via JWT et OAuth2.
Une gestion complète des colis, adaptée au rôle de l’utilisateur.
Une navigation fluide et sécurisée entre les différentes fonctionnalités.
Une consultation claire et synthétique des informations logistiques.
Une réduction des erreurs humaines grâce à des formulaires validés.
Une meilleure visibilité opérationnelle pour les gestionnaires.
